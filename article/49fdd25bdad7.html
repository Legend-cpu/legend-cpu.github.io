nodejs安装及环境配置

Windows

一、进入官网地址下载安装包

https://nodejs.org/zh-cn/download/

选择对应你系统的Node.js版本，如果想下载指定版本，点击【以往的版本】，即可选择自己想要的版本下载

二、安装程序

（1）下载完成后，双击安装包，开始安装Node.js

（2）可自定义安装路径，然后一路next到finish。

（3）测试安装是否成功，在cmd中输入node -v和npm -v看是否有版本号输出，成功显示版本说明安装成功

三、环境配置

（1）找到nodejs的安装目录，在/Node.js根目录下新建两个文件夹【node_global】和【node_cache】

（2）创建完毕后，使用管理员身份打开cmd命令窗口，输入

①npm config set prefix “你的路径\node_global” （复制你刚刚创建的“node_global”文件夹路径）
②npm config set cache “你的路径\node_cache” （复制你刚刚创建的“node_cache”文件夹路径）

路径是使用绝对路径。

（3）配置环境变量

①【此电脑】-单击右键-【属性】-【高级系统设置】-【环境变量】

② 在【系统变量】中点击【新建】：

变量名：NODE_PATH

变量值：刚刚的node_global文件夹路径\node_modules

然后你就会发现【node_global】里多出了一个【node_modules】文件夹。

Tips: 如果输入变量值之后没有自动创建【node_modules】文件夹，就在【node_global】下手动创建一个【node_modules】文件夹，再复制你创建的【node_modules】文件夹的路径地址到变量值

③编辑【用户变量】中的【Path】，将默认的 C 盘下【 AppData\Roaming\npm 】修改成 【node_global】的路径，点击确定

④在【系统变量】中选择【Path】点击【编辑】添加【NODE_PATH】（显示为%NODE_PATH%），随后一直点击【确定】

四、测试

配置完成后，全局安装一个最常用的 express 模块进行测试

npm install express -g // -g代表全局安装

Tips：如果出现安装失败的话可能是你没有使用管理员身份运行cmd窗口，或者可以修改一下【node_global】和【node_cache】的权限

步骤： 鼠标右击【node_global】的文件夹，点击【属性】，再点击【安全】，再点击【编辑】，将权限都勾上，随即点击【确定】即可，【node_cache】步骤同理。

五、安装淘宝镜像

①安装淘宝镜像：npm config set registry https://registry.npm.taobao.org
查看是否成功：npm config get registry

②安装cnpm（按需安装，cnpm不受package-lock.json影响，可能会导致下载npm包时依赖版本不一致的情况，且cnpm安装包也不会自动生成package-lock.json文件，故建议还是使用npm配置国内镜像）

如何发送http请求并接收响应？

方法1：http.request(url, options, callback)

回调函数只自动接收一个res参数，即返回的响应，它是一个http.IncomingMessage对象，该对象为可读流。

http.request()方法本身返回一个http.ClientRequest对象，是一个可写流，发送post请求时需要将post的数据用可写流的write方法写进该对象中。

其中url和options参数都是可选的，但是它们必须要能够确定所要请求的地址，有了url就不必使用options，没有url就必须在options中指定host和path两个字段，如果两个都被指定，options参数优先。

options部分可指定字段：

  • auth，即’user:password’
  • defaultPort：默认端口，当agent.defaultPort被指定时使用该值，否则就采用undefined
  • family：解析host时使用的ip地址族，值为4或6，代表ipv4和ipv6，默认是二者都被使用
  • headers：一个包含多个请求头字段的对象
  • host：所要请求的域名或ip，默认为’localhost’
  • localPort：要连接的本地端口
  • method：http请求方法，默认为’GET’
  • path：域名后面的路径，可包含query参数，默认为’/‘
  • port：远程连接服务器的端口，当defaultPort被指定时使用该值，否则就使用80端口
  • timeout：设置超时，单位为毫秒

常用的是headers、host、method、path和port，auth和timeout次之。

http.request()方法需要与其返回对象的end方法搭配使用，这样请求才能发送出去。

另外请记得在http.request()方法返回的对象上用on方法注册一个error事件监听器来监听可能出现的错误。

nodejs中的流

流的基本使用

nodejs中总共有四种流，比较常见的是三种：writeable可写流，readble可读流，还有duplex可读&可写流。

对于可写流，我们通常会关注调用它的write方法和end方法向其中写入数据，当流或其依赖资源被关闭后会触发’close’事件，当调用end方法写入所有数据后会触发’finish’事件。

对于可读流，我们通常会关注监听它的data事件和end事件，监听data事件可用于读取可读流中的数据，监听end事件可用于在读取完成后进行某操作。我们还会关注使用它的pipe方法、read方法、setEncoding方法、unpipe方法。

readable.pipe(writable)：首先要有一个可读流和一个可写流，然后调用可读流的pipe方法并传入可写流为参数可以将可读流转换为flow模式，并将其中的数据推入到可写流中，该方法会返回写入数据后的可写流。

或者也可以链式调用，即src.pipe(dest1).pipe(dest2)。pipe方法是数据从流到流的转移。

readable.read(size)：读取可读流中的数据并返回，如果没有用readable.setEncoding方法设置编码，默认将数据以Buffer对象形式返回。size参数是可选的，单位为字节，最大不得超过1GB。注意该方法只应该在可读流处于pause模式中才可调用，flow模式中该方法会被自动调用直到缓冲区中的数据读取完毕。

注：可读流是从缓冲区中读取数据，可写流是将数据先写入缓冲区，注意缓冲区的大小问题。

缓冲区：又称缓存，是一块内存区，CPU将处理完的数据写入外部设备时先写入缓冲区，外部设备在合适的时候去一次性读取；CPU向外部设备读取数据时先读入缓冲区，CPU在合适的时候去一次性读取。作用：使高速的CPU和低速的外部设备协调运作，降低用户态和内核态的转换次数，提高运行速度。

readable.setEncoding()：设置从可读流中读取数据的编码方式，默认无编码，直接返回Buffer对象；设置编码会使数据以指定编码的字符串返回，比如’utf8’或者’hex’。

readable.unpipe(dest)：解除之前被pipe方法连接起来的可读流和可写流，如果不指定dest参数则所有被pipe连接的流都将解除联系。

nodejs 的模块加载方法

概述

JavaScript 现在有两种模块。一种是 ES6 模块，简称 ESM；另一种是 CommonJS 模块，简称 CJS。

CommonJS 模块是 Node.js 专用的，与 ES6 模块不兼容。语法上面，两者最明显的差异是，CommonJS 模块使用require()和module.exports，ES6 模块使用import和export。

它们采用不同的加载方案。从 Node.js v13.2 版本开始，Node.js 已经默认打开了 ES6 模块支持。

Node.js 要求 ES6 模块采用.mjs后缀文件名。也就是说，只要脚本文件里面使用import或者export命令，那么就必须采用.mjs后缀名。Node.js 遇到.mjs文件，就认为它是 ES6 模块，默认启用严格模式，不必在每个模块文件顶部指定"use strict"。

如果不希望将后缀名改成.mjs，可以在项目的package.json文件中，指定type字段为module。

一旦设置了以后，该项目的 JS 脚本，就被解释成 ES6 模块。

如果这时还要使用 CommonJS 模块，那么需要将 CommonJS 脚本的后缀名都改成.cjs。如果没有type字段，或者type字段为commonjs，则.js脚本会被解释成 CommonJS 模块。

总结为一句话：.mjs文件总是以 ES6 模块加载，.cjs文件总是以 CommonJS 模块加载，.js文件的加载取决于package.json里面type字段的设置。

注意，ES6 模块与 CommonJS 模块尽量不要混用。require命令不能加载.mjs文件，会报错，只有import命令才可以加载.mjs文件。反过来，.mjs文件里面也不能使用require命令，必须使用import。

加载路径

ES6 模块的加载路径必须给出脚本的完整路径，不能省略脚本的后缀名。import命令和package.json文件的main字段如果省略脚本的后缀名，会报错。

为了与浏览器的import加载规则相同，Node.js 的.mjs文件支持 URL 路径。

上面代码中，脚本路径带有参数?query=1，Node 会按 URL 规则解读。同一个脚本只要参数不同，就会被加载多次，并且保存成不同的缓存。由于这个原因，只要文件名中含有:、%、#、?等特殊字符，最好对这些字符进行转义。

目前，Node.js 的import命令只支持加载本地模块（file:协议）和data:协议，不支持加载远程模块。另外，脚本路径只支持相对路径，不支持绝对路径（即以/或//开头的路径）。

内部变量

ES6 模块应该是通用的，同一个模块不用修改，就可以用在浏览器环境和服务器环境。为了达到这个目标，Node.js 规定 ES6 模块之中不能使用 CommonJS 模块的特有的一些内部变量。

首先，就是this关键字。ES6 模块之中，顶层的this指向undefined；CommonJS 模块的顶层this指向当前模块，这是两者的一个重大差异。

其次，以下这些顶层变量在 ES6 模块之中都是不存在的。

  • arguments
  • require
  • module
  • exports
  • __filename
  • __dirname
