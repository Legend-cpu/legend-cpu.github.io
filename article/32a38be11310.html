npm包管理器介绍

安装所有依赖

如果一个项目有一个package.json文件，运行npm install就可以安装项目所需要的一切依赖到node_modules文件夹里，如果没有这个文件夹就会创建一个。

安装单个包

运行npm install <package-name>安装指定的单个包。并且从npm 5开始这个命令会将包名添加到package.json文件的dependencies对象里，而在版本5之前，这样做需要在该命令后增加一个标志--save（简写为-S）才行。

其他标志：

  • --save-dev将安装包并添加到package.json文件的devDependencies对象中，可简写为-D
  • --no-save将安装包并添加到package.json文件的dependencies对象中
  • --save-optional将安装包并添加到package.json文件的optionalDependencies对象中，可简写为-O
  • --no-optional将阻止可选的依赖被安装。

devDependencies 和 dependencies 之间的区别在于前者包含开发工具，如测试库，而后者与生产中的应用程序捆绑在一起。

如果可以使用依赖项，但如果找不到或安装失败，您希望 npm 继续，那么您可以将其放在 optionalDependencies 对象中。运行npm install --no-optional将阻止安装这些依赖项。

在空文件夹下运行npm install <package-name>将自动在当前目录下创建node_modules文件夹、package.json文件和package-lock.json文件。

更新包

运行npm update可以检查所有包的更新并更新需要更新的包。也可以指定包名单独更新某个包。

版本控制

除了普通下载之外，npm还支持版本控制，因此您可以指定软件包的任何特定版本，或者要求一个高于或低于您需要的版本。

指定库的显式版本还有助于让每个人都使用相同版本的包，以便整个团队运行相同的版本，直到package.json文件更新。

运行任务

npm run <task-name>

在package.json文件里有一个scripts对象，其属性键是任务名称，属性值就是要运行的命令字符串。这样就可以将又长又复杂的命令封装为一个简单的任务名，再通过npm run <task-name>来运行。

npm在哪里安装包？

npm安装包时位置有两种，一种是局部安装，即在当前目录下的node_modules文件夹里安装；另一种是全局安装，即在你配置到node_modules环境变量里的那个路径的node_modules文件夹里安装。npm install <package-name>是局部安装方式，全局安装需要使用npm install -g <package-name>。

使用npm root -g命令可以查看哪个位置被配置为了node_modules环境变量。

一般来说我们推荐使用局部安装的方式安装包，这样能保证我们的电脑上能够有多个应用，且每个应用能够拥有自己版本的包；如果使用全局安装，会使得所有项目都使用同一个版本，会导致维护上的困难。但是当一个包要安装来提供在命令行运行的命令或者在多个项目中复用时应使用全局安装的方式。

如何用npm使用或执行一个包

在代码中使用安装的包：const _ = require('lodash')

如果包是可执行文件的话，一般会被放在node_modules/.bin/文件夹里，从npm 5.2开始其内置了一个npx工具，只需要运行npx <package-name> argv命令即可运行。

package.json文件

package.json文件可以当作是一个配置文件，里面是一个json对象，对象中的属性含义如下：

  • version：当前版本
  • name：应用程序/包的名称
  • author：应用程序/包的作者名字，可以附带上邮箱和个人网站网址
  • contributors：贡献者名单
  • bugs：多数时候是一个指向Github issues页面的链接
  • homepage：应用程序/包的官网首页地址
  • license：比如MIT
  • keywords：应用程序/包相关的关键词
  • description：应用程序/包的简要说明
  • repository：比如github仓库的地址
  • main：应用程序入口文件
  • private：设置为true可以防止应用程序/包被意外发布到npm
  • scripts：一组可以运行的脚本
  • dependencies：npm设置作为依赖项安装的包列表
  • devDependencies：npm设置作为开发依赖项安装的包列表
  • engines：应用程序/包适用于哪些版本的Node.js、npm及yarn
  • browserslist：应用程序/包要支持哪些浏览器及其版本

package-lock.json文件

package-lock.json 的作用是锁定依赖结构即安装的包的版本号，即只要你目录下有 package-lock.json 文件，那么你每次执行 npm install 后生成的 node_modules 目录结构一定是完全相同的。

npm依赖项和开发依赖项

一个项目有开发阶段和投入生产阶段，依赖项是开发阶段和生产阶段都需要的包，而开发依赖项是开发阶段需要，但生产阶段不需要的包，如测试包、webpack、babel等。

在使用npm install安装包时会自动安装为依赖项并添加到package.json文件的dependencies列表中；添加-D或者--save-dev参数会安装为开发依赖项并添加到devDependencies列表中。

如果项目中有一个package.json文件，使用npm install安装时默认为开发环境，会安装依赖项和开发依赖项；如果项目已投入生产需要避免安装开发依赖项，可添加--production参数来避免安装。

查找已安装的 npm 包版本

使用npm list命令查看当前文件夹下的已安装npm包，用npm list -g命令查看全局已安装npm包。

npm list <package>查看特定包的版本。

npm list --depth=<int>来获取直到指定层级的包和依赖。

查看npm存储库中包的最新可用版本：npm view <package> version

安装npm包的历史版本

用@语法来指定安装包的特定版本：npm install <package>@<version>

查看软件包的所有历史版本：npm view <package> versions

更新包的版本

使用npm update将package.json中的包更新到最新的补丁和次要版本，但不包括主要版本。

使用npm outdated查看存储库中过时的软件包。

使用npm-check-updates包将package.json中的包全部更新到最新版本：先全局安装npm-check-updates包，然后运行ncu -u命令，最后运行npm install安装。

npm中的语义版本控制规则

如果 Node.js 包中有一件很棒的事情，那就是他们都同意使用语义版本控制来进行版本编号。

语义版本控制的概念很简单：所有版本都有 3 位数字： x.y.z.

  • 第一个数字是主要版本
  • 第二个数字是次要版本
  • 第三个数字是补丁版本

当你发布一个新版本时，你不只是随心所欲地增加一个数字，而是你有规则：

  • 当您进行不兼容的 API 更改时升级主版本
  • 当您以向后兼容的方式添加功能时，您升级了次要版本
  • 当您进行向后兼容的错误修复时，您会升级补丁版本

该约定在所有编程语言中都采用，并且非常重要的是，每个 npm包坚持它，因为整个系统都依赖于它。

为什么这么重要？

因为 npm设置一些我们可以在 package.json文件来选择它可以更新我们的包的版本，当我们运行 npm update.

规则使用这些符号：

  • ^
  • ~
  • >
  • >=
  • <
  • <=
  • =
  • -
  • ||

让我们详细看看这些规则：

  • ^：它只会做不改变最左边的非零数字的更新，即次要版本或补丁版本可能会发生变化，但主要版本不会发生变化。 如果你写 ^13.1.0, 运行时 npm update，它可以更新为 13.2.0, 13.3.0甚至 13.3.1, 13.3.2等等，但不是 14.0.0或以上。
  • ~: 如果你写 ~0.13.0跑步时 npm update它可以更新到补丁版本： 0.13.1可以，但是 0.14.0不是。
  • >：您接受任何高于您指定的版本
  • >=：您接受等于或高于您指定版本的任何版本
  • <=：您接受任何等于或低于您指定的版本
  • <：您接受任何低于您指定的版本
  • =: 你接受那个确切的版本
  • -：您接受一系列版本。 例子： 2.1.0 - 2.6.2
  • ||: 你组合集合。 例子： < 2.1 || > 2.6

您可以组合其中一些符号，例如使用 1.0.0 || >=1.1.0 <1.2.0使用 1.0.0 或 1.1.0 以上的一个版本，但低于 1.2.0。

还有其他规则：

  • 无符号：您只接受您指定的特定版本（ 1.2.1)

  • latest：您想使用可用的最新版本

卸载npm包

使用npm uninstall <package>命令从项目根文件夹卸载包，如果是全局包要加-g参数，如果加-S或者--save参数还会删除package.json文件中对该包的引用。

package.json中的dependency和devDependency会在包卸载后自动更新。

npx 包运行器

npx允许您运行使用 Node.js 构建并通过 npm 注册表发布的代码。

npx使得人们可以在不先安装命令的情况下运行该命令，并且仍然可以使用@语法运行同一命令的不同版本。

npx甚至可以直接从url运行代码：npx <url>。
