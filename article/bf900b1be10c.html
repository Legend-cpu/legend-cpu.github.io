npm托管静态资源使用方法

发表于2022-10-17|更新于2022-10-19|博客建设
|字数总计:2.1k|阅读时长:6分钟

前言

自己的服务器带宽太小，网站加载一个500多KB的图片/字体等静态资源都需要十秒左右的时间, 大大拖慢了网站加载的速度。说套CDN吧，国内CDN基本都是按流量计费的，对于比较吝啬的我来说还是白嫖比较放心，之前在糖果屋看到了这么一篇教程，就跟着做了一遍，在此把过程记录下来。

步骤

首先要在npm官网上注册一个自己的账号，并完成邮箱的验证。

本地肯定是要安装一个Nodejs的，搞hexo的肯定都有。

然后在本地合适的地方新建一个文件夹作为npm包的本地源文件，假设命名为pkg。然后我们要先做一些准备工作。

首先在pkg目录下打开命令行执行 npm config set registry https://registry.npmjs.org命令将源切换为官方源。国内的npm用户多数会在用的时候把这个源设置为淘宝镜像源，但是我们是要把我们的包往npm官网上发而不是发到淘宝的镜像库中，所以要把这个源切换回官方源。如果以后要下载npm包的时候可以再把源切换为淘宝镜像源。（如果觉得切换很麻烦的话或许可以忽略本文了）

然后在pkg目录下执行 npm init命令来将目录初始化为一个npm包，命令行中将会打印出一串信息，即将要生成的package.json里的内容，一路回车确认即可，然后目录底下就会生成一个package.json文件。

因为你每次包的内容有改动然后你把新的包发布上去的时候一定要修改package.json文件内的version字段，即包的版本号，否则无法发布。而我们肯定是不想每次改版本就得改一次引入链接的，所以cdn链接中的version部分可以用 latest代替，表示最新版本。

刚说到需要更新版本号的问题，你可以手动修改package.json文件里的版本号，或者你可以在包的目录下打开命令行运行 npm version patch命令来更新版本号。

npm包的版本号有三位，是x.y.z这样的形式，且xyz都是非负整数。x称为major，即主版本号；y称为minor，即次版本号；z称为patch，即补丁版本号。x只应当在包还是开发版的时候取0，一般x从1开始，y、z可以从0开始。x位应当在项目进行大改并且向下不兼容的时候更新，增加1即可；y位应当在项目进行向下兼容的改动的时候更新，增加1即可；z只应当在项目进行一些bug fix之类的工作的时候更新，增加1即可。

npm version命令可用来自动更新版本号，主要会用到三个： npm version major使主版本号加1， npm version minor使次版本号加1， npm version patch使补丁版本号加1。当然如果你想进行一个大版本更新还是要手动修改package.json文件里的版本号的。

注意，如果npm包的目录同时也是一个git仓库的话， npm version命令将会在更新版本号之后自动进行一次commit并打一个相应版本号的tag。

借助Github Action来发布npm包

其实上面就是使用npm托管静态资源的全部步骤了，但是有时候可能会发现一个问题：执行 npm publish的时候速度很慢，甚至长时间没有响应。这是因为发布包所用的官方源下载包和长传包的速度在国内都是比较慢的。要解决这个问题，我们可以借助于Github Action来执行 npm publish命令，因为Github的仓库服务器在国外，用它来发布包的速度是很快的。
